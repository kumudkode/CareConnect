# Research Design — CareConnect Research Paper

## Type: Exploratory + Descriptive + Applied Research

### Exploratory Research
Identified the gap in accessible AI-powered healthcare tools for patients in India through literature review of existing apps (Practo, 1mg, HealthifyMe) and academic papers on medication non-adherence (WHO, 2003).

### Descriptive Research
Documented the complete system architecture, user workflows, data flows, and feature implementation of CareConnect across all modules.

### Applied Research
Developed and deployed a functional prototype (live at careconnectweb.vercel.app), tested with 30 real users, and measured usability outcomes.

---

## Development Methodology: Iterative Prototyping

| Phase | Activity | Output |
|-------|----------|--------|
| Phase 1 | Requirement gathering, literature review | Problem statement, SRS |
| Phase 2 | Backend services, Firebase setup | Auth, Firestore, storage layer |
| Phase 3 | AI integration (Gemini/Groq) | API routes, Maya assistant |
| Phase 4 | UI design and page development | 12 functional screens |
| Phase 5 | Testing and evaluation | Test results, user survey |
| Phase 6 | Deployment | Live production URL |

---

## Type of Data Used

**Primary Data**
- User feedback (30-participant survey)
- Functional test results (pass/fail per feature)
- AI response quality logs

**Secondary Data**
- WHO medication adherence statistics
- Published AI in healthcare research papers
- Firebase, Groq, and Next.js technical documentation
- Competitor app feature comparisons
